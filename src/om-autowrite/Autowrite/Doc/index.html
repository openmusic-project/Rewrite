Version 3.1 (February 2005) of Autowrite is available!

What is Autowrite

Autowrite is a project carried out at the Laboratoire Bordelais de Recherche en Informatique (LaBRI) of the Université Bordeaux I, by the LLA (Logic, Formal Languages and Applications) team. Autowrite is an experimental tool written in CMU Common Lisp for checking properties of term rewrite systems. It was initially designed to check call-by-need (CBN) properties of TRSs. For this purpose, it implements the tree automata constructions used in the decidability proofs of the call-by-need properties and many useful operations on terms, TRSs and tree automata. This new version of Autowrite includes new functionalities related to TRSs, timing information but the most interesting new feature is the possibility to handle directly (load, save, combine) bottom-up tree automata. The graphical interface (still under construction) running under X11 is written using FreeCLIM , the free implementation of the CLIM specification . From this interface, one can handle TRSs, tree automata and build many tree automata related to TRSs. one can check membership to CBN for the different approximations of a given system.

Main features of Autowrite

The principal features of Autowrite are:

  • Terms and term rewrite systems. Given a term t and a term rewrite system R over a signature F
    • check whether: R is left-linear, overlapping, orthogonal, constructor
    • check whether: R is forward-branching
    • compute approximations of R (strong-approximation, nv-approximation, linear-growing-approximation, growing-approximation)
    • decide whether R is in CBN_alpha (for the different approximations alpha) and exhibits an alpha-free term in the negative case.
    • Check whether NF(R) (the set of normal forms) is empty
    • Check whether ENF(R) (the set of external normal forms) is empty
    • If F is the signature consisting of the terms appearing in R and G be F extended by a fresh constant symbol, decide whether the set of terms in T(F) normalizable with (R,F), WN(R,F), is different (strictly included) into the set of terms in T(F) normalizable with (R,F), WN(R,G,F). The same check can be done with R extended with the rule o -> o.
    • Check whether the current approximation S is arbitrary i.e. if there exists a term t in T(F) such that t -*->(R,FU{@}) @
  • Tree automata. Given tree automata A and a term t
    • decide whether t is in L(A)
    • decide whether L(A) is empty
    • compute a deterministic version of A
    • compute a reduced version of A
    • compute the complement of A
    • given another automaton B compute the union or the intersection of A and B
  • Automata and term rewrite systems.
    • given a set of ground terms S: compute a tree automaton A_S such that L(A_S) = S
    • given a set of linear terms S: compute a tree automaton A_S such that L(A_S) recognizes the set of instances of S
    • given the set of left-handsides of a left-linear TRS R: compute an automaton A_NF(R) such that L(A_NF(R)) = NF(R)
    • given tree automaton A and a left-linear growing term rewrite system:
      compute an automaton C_[R,A] such that L(C_[R,A]) = (-*->)[L(A)]
    • given a left-linear growing extended term rewrite system R: compute an automaton D_R such that L(D_R) is empty if and only if R is in CBN.

Obtaining Autowrite

The Autowrite distribution is publically available for research purposes. The Autowrite distribution consists of a Linux gzip-compressed tar file.
The currently supported machines and operating systems are listed below.

  • PC x86 under Linux Autowrite.tgz
    • Other architectures can be made available upon request.

    A collection of examples of term rewrite systems to use with Autowrite is included in the Data directory of the distribution.

    Installing Autowrite

    Download the relevant Autowrite.tgz tar file

    • tar xzvf Autowrite.tgz
    • cd Autowrite
    • ./INSTALL

    Running Autowrite

    • xhost +
    • ./autowrite
    At any time from the command line, the user may get help about the current possibilities by clicking on the right-button. Commands are accessible either thru the command line (using completion) or from the menus items. For instance, instead of hitting the Quit button you may type Q ; instead of selecting the Load spec of the File menu, you may type 'Lo Sp '...
    Note that some examples may run for a very long time as the problem of deciding membership to CBN_alpha (can be from exponential to triply exponential).
    Add your own specifications in the Data directory

    Please refer to the Autowrite user's guide to see how to use Autowrite. Also available in Postscript or PDF
